### Support for 100 MB Document

1.  If the send attachment API is invoked, the socket timeout will default to 20 seconds. However, this can be customized in im.properties using the key "zohoim.attachment.http.socket.timeout.ms".
2.  By default, attachments up to 50 MB are supported, but this limit can be increased to a maximum of 100 MB upon customer request for both incoming and outgoing IM messages.
3.  In Local Zoho, sending 100 MB files is not possible because attachments are streamed using an internal framework called AWS, which does not support such large file streams.

### Send Attachment to a Session

```
//Message to be send
String messageToSend= "Here is your payment bill";

// constrcut IMAttachment Object
IMAttachment attachment = new IMAttachment.Builder(ATTACHMENT_NAME, ATTACHMENT_INPUT_STREAM)
      .setContentType(ATTACHMENT_CONTENT_TYPE)
      .setContentLength(ATTACHMENT_SIZE)
      .build();

//send attchment message through IM
IMMessage sentMessage = ZohoIM.getInstance(IM_ORG_ID, SERVICE_ORG_ID)
     .getMessageAPI(USER_ID)
     .sendAttachment(IM_SESSION_ID, null, messageToSend, attachment);
```
