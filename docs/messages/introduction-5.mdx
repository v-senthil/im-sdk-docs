The message entity is a child entity of the session entity, and it is possible to have multiple messages within a single session. A message refers to a unit of communication sent by one user to another user. It could be a text-based message, an image, a video, an audio recording, or any other form of media

Postman Collection: [link](https://documenter.getpostman.com/view/18635004/2s935vkfHz#8ed4dc0c-6a20-4da3-9a6f-818712562828)

OAuth Scopes: ZohoIM.messages.ALL

Message can be sent only if an agent picks up an chat. This can be done by updating the session using agent zuId.

Once an outgoing message is triggered, the message will be first QUEUED to send and then it gets SENT, DELIVERED, READ or FAILED accordingly and the status for each message can be checked in the "status" field of the IMMessage instance.
