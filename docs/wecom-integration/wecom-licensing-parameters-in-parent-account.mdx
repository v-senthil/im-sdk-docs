To provide better clarity to the service teams regarding the WeCom account's license and trial status, we are now including the necessary details in the parent account parameters itself. Using these parameters, the service team can:

-   Inform the customer when to purchase a license.
-   Notify the customer when their trial is about to expire.
-   Identify the current status of the license.

### 1\. TRIAL\_START\_TIME

-   Represents the start time of the WeCom account trial configured in IM.
-   This is the timestamp when the customer first authorized their WeCom account in Zoho IM.

### 2\. TRIAL\_END\_TIME

-   Represents the end time of the trial period.
-   By default, WeCom provides a 90-day trial from the trial start time.
-   The trial end time is recorded in this parameter.

### 3\. LICENSE\_STATUS

-   Represents the current license status of the WeCom account in IM.
-   Possible values:

### 4\. LICENSE\_ACTIVATED\_TIME

-   The timestamp when a purchased license was activated.

### 5\. LICENSE\_EXPIRY\_TIME

-   The timestamp when the purchased license expires.

### 6\. CORP\_NAME

-   By default, the corporate name of the parent WeCom account is stored in this parameter.

## Notes

-   All parameters are optional except CORP\_NAME.
-   Trial-related parameters (TRIAL\_START\_TIME and TRIAL\_END\_TIME) are available only if the customer is in trial.
-   License-related parameters (LICENSE\_ACTIVATED\_TIME and LICENSE\_EXPIRY\_TIME) are available only if the license is purchased via IM Credits.
-   If no license or trial information is available, the LICENSE\_STATUS will be set to NONE.
-   Service teams can access these details by invoking the Parent Accounts API.
-   All of the time is available in GMT.
