### Pre-Approved Message Templates

A message template is required to start a business-initiated conversation.

-   Business-initiated messages can only be sent to users who have opted-in to receive messages from your business and must be transactional.
-   When sending notifications, one must use a previously approved template.

### Template Message in IM:

-   Each Template Message will be created under a channel only (i.e under the WABA which mapped to that channel). If channel is not given, template message will be added under default WABA.
-   If there is a channel update where in, the channel is updated to different WABA, the Template message already present under the channel before update will not be migrated. After update, you have to create new templates.
-   Supported Template Tags (category) - [link](https://documenter.getpostman.com/view/18635004/2s935vkfHz#32f06200-7134-4fe0-b9fb-17c21a733852)
-   Supported Template Languages - [link](https://documenter.getpostman.com/view/18635004/2s935vkfHz#70d33de6-cf17-499d-9bc7-88f7068ae480)

### Header

1.  Optional
2.  Supported Types:

1.  TEXT

1.  60 characters
2.  1 placeholder
3.  There should be a space before the placeholder
4.  No new lines, formatting characters, emojis or asterisks

2.  IMAGE

1.  Supported formats: jpg, jpeg,png
2.  Maximum Size: 5MB

3.  VIDEO

1.  Supported formats: mp4,3gpp
2.  Maximum Size: 16MB

4.  DOCUMENT

1.  Supported formats: pdf
2.  Maximum Size: 40MB

### Body

1.  Mandatory
2.  1024 characters (including the replaced placeholders values)
3.  A Maximum of 10 placeholders is allowed.
4.  There should be a space before placeholders.
5.  The message body can't have more than two consecutive newline characters.
6.  Cannot have only parameters
7.  Cannot have more than 10 emojis.

### Footer

1.  Optional
2.  60 characters
3.  No placeholder
4.  No newlines or emojis or special characters

### Buttons

1.  Optional
2.  Max 10 buttons
3.  Support Types:

1.  QUICK REPLY

1.  20 characters
2.  Message cannot be repeated
3.  No placeholders

2.  PHONE NUMBER

1.  Message should have 20 characters
2.  The phone number should have 20 characters only.
3.  It should be a valid phone number
4.  No placeholders

3.  URL

1.  Message should have 20 characters
2.  The URL can have 1000 characters (including the placeholder replaced value)
3.  It should be a valid URL starting with http or https. Base URL is mandatory.
4.  Can contain one placeholder

4.  Button Formation combination can be in any of the following:

1.  10 QUICK\_REPLY buttons
2.  8 QUICK\_REPLY buttons, 1 PHONE\_NUMBER, and 1 URL
3.  1 PHONE\_NUMBER
4.  1 URL
5.  1 PHONE\_NUMBER and 1 URL
